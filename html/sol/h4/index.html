Say we are going to search the first primes, if we used 5 nested for loops, or a recursion stack with 5 frames, then the code will spend a lot of time searching the higher primes. So we are going to search using a breath-first method instead of depth-first. Start by checking if every constant is the smallest prime, . If not, we are going to introduce the next prime, . Assume the first constant is the new prime and search through all smaller primes for the other constants. Do the same thing for the next constant, until all the constants have been searched using this new prime. If the function still doesn't matter, repeat with the next prime. By continuing this algorithm, it will check all of the smaller prime combinations before moving onto a larger prime.
